Privacy Policy - Always Fresh

Last updated: 04/14/2026

This Privacy Policy explains how Always Fresh accesses, uses, stores, and protects information related to the use of the mobile application.

1. Controller and contact

Application: Always Fresh

Privacy and support contact: vikena.smart@gmail.com

2. Data handled by the app

Always Fresh is designed to work primarily with data stored on the user's device. The app may process the following categories:

  • User-entered data: product name, brand, description, category, location, quantity, price, barcode or QR code, and expiration date.
  • Local technical data: selected language, reminder time, notification status, and folder paths chosen by the user for backups.
  • Camera-related data: barcode and QR code scanning when the user voluntarily starts the scanner.
  • Backup files: JSON files exported or restored by the user containing inventory information and remembered products.

3. Permissions and purpose

  • Camera: used only to scan barcodes and QR codes when the user requests it.
  • Notifications: used to send local reminders about products nearing expiration, only if the user enables the feature.
  • Access to user-selected files: used to import or export backups manually through the file or folder picker.
  • Internet: used to open external legal links and, if the user uses features that depend on developer-configured external services, to send HTTPS requests.

4. How data is used

  • Display and organize the food inventory.
  • Detect products nearing expiration and generate local reminders.
  • Remember information associated with scanned codes to speed up future entries.
  • Create and restore local backups requested by the user.
  • Apply language and settings preferences inside the app.

5. Storage and retention

Inventory information and settings are stored locally on the user's device. Automatic and manual backups remain available until the user deletes them, uninstalls the app, or overwrites them.

6. Data sharing

Always Fresh does not sell personal data. The app does not automatically share inventory information with third parties for advertising purposes.

Data may be transferred only in these situations:

  • When the user voluntarily exports or shares a backup file.
  • When the user opens an external link.
  • When an optional feature configured by the developer uses an external translation service or version policy service over HTTPS.

7. Third-party services

The application may integrate libraries or technical services needed for local device functionality, such as notifications, file picking, file sharing, or code scanning. If an external translation or version policy service is configured, text sent to that service will be processed according to that provider's terms and privacy practices.

8. Security

Reasonable measures are applied to protect the information handled by the app. However, no storage or transmission method is completely fail-safe, and the user is responsible for protecting access to their device and backup files.

9. Children

The app is not specifically directed to children and is not intended to knowingly collect personal data from children.

10. User controls

  • The user can edit or delete inventory items from inside the app.
  • The user can disable notifications from the app or device settings.
  • The user can delete backup files created manually or automatically.
  • The user can uninstall the app to remove locally managed app storage, except for files exported outside the app's private storage area.

11. Changes to this policy

This Privacy Policy may be updated to reflect legal, technical, or functional changes. The current version will always be the one published on this page.

12. Contact

If you have questions about privacy, data handling, or requests related to the app, contact: vikena.smart@gmail.com
